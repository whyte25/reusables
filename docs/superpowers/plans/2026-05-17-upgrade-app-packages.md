# Upgrade App Packages Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Upgrade the documentation app stack to the latest compatible Fumadocs, Tailwind CSS, shadcn CLI/component conventions, Next.js, and React versions without burying breaking changes in one diff.

**Architecture:** Upgrade in dependency layers: framework/runtime first, Tailwind CSS v4 configuration second, Fumadocs v16 API changes third, shadcn/component cleanups fourth. Each layer ends with install, typecheck/build, and a small commit so failures can be isolated.

**Tech Stack:** Bun, Next.js, React, TypeScript, Tailwind CSS, shadcn/ui, Fumadocs, Changesets.

## Documentation Findings

- Tailwind v4 requires modern browsers and Node.js 20+, replaces `@tailwind` directives with `@import "tailwindcss"`, moves PostCSS integration to `@tailwindcss/postcss`, and no longer auto-detects JS config files.
- Fumadocs v15+ expects Tailwind v4 CSS imports. Fumadocs theme setup now uses CSS imports such as `fumadocs-ui/css/shadcn.css` and `fumadocs-ui/css/preset.css`.
- Fumadocs v16 removes deprecated APIs. In this repo, known affected imports are `fumadocs-ui/provider`, `fumadocs-core/server`, and likely `fumadocs-ui/page`.
- Fumadocs v16 requires React `19.2.0+`; Next projects using Fumadocs UI must use Next `16+`.
- shadcn Tailwind v4 migration includes CSS variable conversion to `@theme inline`, dependency bumps, optional `w-* h-*` to `size-*`, deprecated `tailwindcss-animate` replacement, and optional component overwrite from `shadcn@latest`.
- shadcn registry latest still builds from a root `registry.json`; each item must conform to the latest `registry-item.json` schema. The current repo already has `registry.json`, `shadcn build`, and a custom `scripts/build-registry.mts` post-processor, so the migration must verify generated `public/r/*.json` item payloads, not only `components/ui`.
- Next 16 removes `next lint`; `package.json` currently has `"lint": "next lint"`, so that script must change.

## File Structure

- Modify `package.json`: upgrade root dependencies, replace `next lint`, remove obsolete Tailwind v3 packages if confirmed unused.
- Modify `packages/notify/package.json`: upgrade package-local React types, `tailwind-merge`, `lucide-react`, and motion dependency if keeping publishable package compatibility.
- Modify `bun.lock`: regenerated by `bun install`.
- Modify `postcss.config.js`: switch from `tailwindcss` and `autoprefixer` plugins to `@tailwindcss/postcss`.
- Modify `app/global.css`: migrate Tailwind v3 directives and theme values to Tailwind v4 CSS-first config with Fumadocs and shadcn imports.
- Remove or stop relying on `tailwind.config.js`: keep temporarily only if `@config` is needed during migration; delete once CSS-first config fully covers custom tokens.
- Modify `components.json`: update shadcn config for Tailwind v4 and latest registry aliases if CLI migration changes schema/style defaults.
- Modify `registry.json`: update registry metadata/item shape to latest schema if validation/build reports deprecated fields.
- Modify `scripts/build-registry.mts`: keep custom generated index/import rewrite compatible with latest `shadcn/registry` types and latest generated item payloads.
- Modify `components/provider/provider.tsx`: change `RootProvider` import to `fumadocs-ui/provider/next`.
- Modify `app/docs/[[...slug]]/page.tsx`: change `getGithubLastEdit` import from `fumadocs-core/server` to `fumadocs-core/content/github`; adjust page component imports if v16 no longer exports from `fumadocs-ui/page`.
- Modify `components/code-block.tsx`: change `highlight` import away from `fumadocs-core/server` if v16 build reports it removed.
- Modify class names in `registry/`, `components/`, `packages/notify/src/`, and `app/`: replace removed Tailwind utilities such as `bg-opacity-*`, `flex-shrink-*`, and any renamed v4 utilities reported by build or visual review.

## Risk Register

- Fumadocs v16 may require additional import moves beyond the ones visible from static search.
- Next 16’s Turbopack-by-default build may expose webpack-only assumptions if any are hidden in config or dependencies.
- Tailwind v4 browser support is stricter: Safari `16.4+`, Chrome `111+`, Firefox `128+`. Do not proceed if the site must support older browsers.
- Overwriting shadcn components can erase local customization. Use `diff` first, then overwrite only low-customization components.
- `packages/notify` is publishable; React peer compatibility should stay broad unless a deliberate major package release is planned.
